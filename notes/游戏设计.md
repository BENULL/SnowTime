# 角色设定
你是一个资深的全栈游戏开发工程师，精通 React, Node.js, Socket.io 以及响应式网页设计。你的任务是根据我提供的桌游《Snow Time》的规则，从零开始开发一款支持在线多人联机、可创建房间、且兼容PC和手机浏览器分辨率的Web端桌游。

# 技术栈要求
- **前端:** React (使用 Vite 构建), Tailwind CSS (用于响应式布局), Socket.io-client
- **后端:** Node.js, Express, Socket.io (用于处理房间逻辑和实时游戏状态同步)
- **状态管理:** 后端作为单点数据源 (Single Source of Truth)，前端仅负责渲染和发送动作指令。

# 游戏核心系统设计

## 1. 房间与联机系统
- 用户进入网站后可以选择“创建房间”或“加入房间（输入房间号）”。
- 玩家需要输入昵称。
- 房间内支持 3-5 名玩家。房主可以点击“开始游戏”。

## 2. 游戏核心规则与逻辑转化 (基于《Snow Time》)
请在后端的 `GameState` 中实现以下完整逻辑：

### 2.1 游戏组件与初始状态
- **计分轨道 (Scoring Track):** 设定为一条包含30个格子的轨道。需要随机或固定分配一些“奖励格(Bonus Spaces)”，分为三种类型：Fruit(水果), Fight(战斗), Mana(法力)。
- **果实 (Fruit):** 游戏总共有15个果实。
- **玩家初始手牌:** 每位玩家发一套专属颜色的10张牌。
  - 7张角色牌 (数字 1-7)。
  - 3张特殊牌: Healer(治疗者), Watcher(观察者), Blizzard(暴风雪)。
- **神圣树 (The Tree):** 共有 7 个层级 (Level 1 到 Level 7)。

### 2.2 回合流程 (Round Phases)
每个回合包含以下严格顺序：

**阶段 1: 放置果实 (Place the Fruit)**
- 后端模拟掷2个六面骰子(1-6)。
- 在对应的树层级(Level)上各放置1个果实（如果掷出双倍，则在该层放置2个）。
- *注意: Level 7 永远不会有果实。如果没有剩余果实，则不掷骰子。如果只剩1个，只掷1个骰子。*

**阶段 2: 出牌 (Play a Card)**
- 所有玩家从手牌中暗扣一张牌（通过 Socket 提交给服务器）。
- 当所有玩家都提交后，进入结算阶段，公开所有卡牌。

**阶段 3: 结算行动 (Resolve Actions - 核心难点)**
**必须严格从上到下 (Level 7 结算到 Level 1) 依次结算：**

- **A. 特殊卡牌结算:**
  - `Watcher`: 出了此牌的玩家在看到其他人出的牌后，再从手牌中选一张打出。(如果有多个Watcher，同时选择并展示)。使用后永久丢弃。
  - `Blizzard`: 回合内所有打出的**角色牌**全部被丢弃。打出Blizzard的玩家，每有一张被丢弃的角色牌，获得1点战斗分(Fight Point)并在计分轨上前进。使用后永久丢弃。
  - `Healer`: 玩家可以从自己的弃牌堆回收最多2张卡牌回手牌。这张卡永远不会被永久丢弃，回合结束时回到手牌。如果玩家手中只有healer，打出后可以把丢弃的所有牌回收到手牌

- **B. 战斗结算 (Fight):**
  - **规则1 (同层冲突):** 如果多个角色在**同一个Level**，他们全部被击败。每位玩家根据击败的其他角色数量获得对应的战斗分（例如3人在同层，每人得2分）。这些卡牌进入弃牌堆。
  - **规则2 (上下层击落):** 如果一个角色在某一层是**孤独的(alone)**，且**紧挨着的下一层**有角色，高层角色将击败低层角色。高层角色获得战斗分站在树上，角色最终回到手牌。低层角色掉落并进入弃牌堆。*(注意: 结算是从上到下级联的，如果L5击败了L4，L4就掉落了，无法再去击败L3)。*

- **C. 收集果实 (Take Fruit Tokens):**
  - 战斗结束后，仍然留在树上的角色，拿走所在层级的所有果实。每拿1个果实获得1点果实分(Fruit Point)，在计分轨前进。拿走的果实放回公共储备区。

- **D. 法力积分 (Mana Points):**
  - 回合结束时，留在树上**位置最低**的那个角色，获得1点法力分(Mana Point)。

- **E. 奖励格判定 (Bonus Spaces):**
  - 玩家得分后在计分轨移动。如果最终停在某一种“奖励格”（如战斗奖励格），**且**该玩家本回合恰好获得了该类型（战斗）的得分，则立即额外前进1~3格。
  奖励格的位置分别如下：
  法力：4\5\6\17\18\19,在第5和18格+2，第19格+3，其余+1
  战斗：9、10、23、24，在10和23格+2，其余+1
  水果：13、14、26、27，在14和26格+2，其余+1

**阶段 4: 回收与下一回合**
- 留在树上的角色卡回到玩家手牌。
- 被击败的角色卡保持在弃牌堆（面朝上）。
- 任何玩家到达计分轨终点则游戏立即结束，该玩家获胜（由于从上往下结算，高层的玩家可能在结算底层前就瞬间获胜）。

# UI/UX 与响应式设计要求 (PC & Mobile 兼容)
为了确保手机端也能完美游玩，前端设计请遵循以下 Tailwind 规范：
1. **主界面布局:**
   - **PC端 (lg及以上):** 采用左右分栏。左侧显示计分轨道和玩家列表，中间显示大树（7个层级及果实/玩家标记），右侧显示自己的手牌和弃牌区。
   - **移动端 (默认设置):** 采用上下布局或标签页切换。顶部显示简化的游戏信息（当前阶段、玩家分数摘要），中间是可滚动的树状图，底部吸底固定显示自己的手牌（采用横向滑动或折叠面板设计）。
2. **交互:** 
   - 卡牌可点击选中并有二次确认“确认出牌”按钮，防止误触。
   - 动画效果：不需要复杂的3D动画，但需要用高亮、卡牌移动的CSS过渡效果来清晰展示“谁打败了谁”、“谁拿到了果实”。
